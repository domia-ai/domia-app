# @domia-app/web — Domia Console

The local Console for a fleet of Domia voice-AI devices: fleet discovery, per-Domia
performance, conversation traces with audio replay, analytics (TTFA, per-stage
latency, eval corpus) and a Chat tester for any flow mode — all served from the
local SQLite the collector mirrors.

## Stack

- **TanStack Start** (Vite + TanStack Router + server functions) — SSR, type-safe routing
- **React 19**, **Tailwind v4**, **shadcn/ui** (Base UI)
- **TanStack Query / Table / Form**, **recharts**
- **Drizzle ORM** + **better-sqlite3** via the `@domia-app/db` workspace package
- **Zod** for env + server-function input validation

## Scripts

```bash
npm run dev      # vite dev on :3000
npm run build    # vite build (Nitro node-server output) + tsc --noEmit
npm run start    # node .output/server/index.mjs (self-contained Nitro server)
npm run lint     # eslint
```

## Layout

- `src/routes/` — file-based routes. `_dashboard.tsx` is the shell (sidebar + header);
  pages live under `src/routes/_dashboard/`. `src/routes/api/` holds raw-`Response`
  server routes (audio proxy/stream, NDJSON export).
- `src/services/` — transport-agnostic business logic (Drizzle reads, mutations, Domia
  node calls). No TanStack Start / React / Request-Response — reusable from a future
  mobile HTTP handler.
- `src/server/` — thin `createServerFn` adapters: validate input with a Zod schema from
  `src/schemas/server.ts`, call the matching service, return the result.
- `src/components/` — `ui/*` are shadcn primitives (managed via `npx shadcn add`, never
  hand-edited); everything else is feature UI.

## Notes

- `src/routeTree.gen.ts` is generated by the TanStack Start plugin and is committed.
- DB path / audio dir / node timeout come from env (`src/config`), server-side only.
- `better-sqlite3` is externalized from the SSR bundle (`vite.config.ts`).
